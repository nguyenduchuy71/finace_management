# Milestone v1.0: FinanceManager MVP

**Status:** ✅ SHIPPED 2026-03-04
**Phases:** 1–6
**Total Plans:** 19

## Overview

Starting from an empty Vite project, this milestone delivered a personal finance dashboard where a user can see all their bank and credit card transactions in one place, organized by billing cycle. Phase 1 laid the data infrastructure. Phase 2 delivered core transaction views with filtering. Phase 3 implemented credit card billing cycle display. Phase 4 added the dashboard charts and polished responsive behavior. Phase 5 added an AI-powered chatbot. Phase 6 optimized performance, added comprehensive testing, polished the UI, and deployed to Vercel.

## Milestone Summary

**Key Decisions:**
- Frontend-only stack (no backend) — API integration points clear, no bottleneck
- MSW deferred render guard pattern — prevents hydration mismatch in development
- Filter state in TanStack Query key — auto-refetch on every filter change, no manual triggers
- Zustand v5 double-curry with useShallow — prevents infinite re-render from object refs
- Anthropic SDK dangerouslyAllowBrowser: true — Anthropic allows browser CORS as of 2025
- Cursor pagination: undefined sentinel — TanStack Query v5 requirement (not null)
- Dark mode FOUC prevention — apply class to documentElement immediately on module load

**Issues Resolved:**
- Vite scaffold in non-empty directory — temp dir workaround
- shadcn requires Tailwind pre-installed — installed Tailwind first
- TypeScript validator reads root tsconfig.json — added compilerOptions.paths to root
- Vitest strict mode — import defineConfig from vitest/config not vite

**Technical Debt Incurred:**
- Real banking API schema unknown — fixtures built to best-guess shape; VND format must be verified
- Billing cycle API contract unknown — assumes statementDate/billingCycleStart/billingCycleEnd fields
- Main bundle 525KB (target was <500KB) — React/TanStack/shadcn shared deps cannot be split further
- Phase 5.1 (Chatbot UI Refinements) planned but never executed — deferred to v1.1
