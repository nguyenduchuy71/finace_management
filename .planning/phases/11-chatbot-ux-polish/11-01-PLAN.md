  <name>Task 3: Enhance ChatMessage with mobile tap-to-reveal pattern for action buttons (Copy, Delete, Regenerate)</name>
  <files>src/features/chatbot/ChatMessage.tsx</files>
  <action>
Enhance ChatMessage.tsx to implement mobile tap-to-reveal for action buttons (Copy, Delete, Regenerate):

1. Add state and hooks:
   - `const [showActions, setShowActions] = useState(false)`
   - `const timeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null)`
   - `useEffect(() => { return () => { if (timeoutRef.current) clearTimeout(timeoutRef.current) } }, [])`

2. Add touch device detection function:
   ```typescript
   const isTouchDevice = () => {
     return (('ontouchstart' in window) ||
       (navigator.maxTouchPoints > 0) ||
       (navigator.msMaxTouchPoints > 0))
   }
   ```

3. Add handleTouchStart function:
   ```typescript
   const handleTouchStart = () => {
     setShowActions(true)
     if (timeoutRef.current) clearTimeout(timeoutRef.current)
     timeoutRef.current = setTimeout(() => setShowActions(false), 4000)
   }
   ```

4. Update the message group div to include touch handler:
   ```jsx
   <div
     className="group flex justify-start mb-3"
     onTouchStart={isTouchDevice() ? handleTouchStart : undefined}
   >
   ```

5. Update the action buttons container CSS to support both hover and tap visibility:
   - **Desktop (hover only):** `opacity-0 group-hover:opacity-100`
   - **Mobile (tap-to-reveal):** `opacity-100` when showActions=true, else `opacity-0`
   - Implementation: Use conditional className based on isTouchDevice()
   ```jsx
   <div className={`flex gap-1 transition-opacity duration-200
     ${isTouchDevice()
       ? (showActions ? 'opacity-100' : 'opacity-0')
       : 'opacity-0 group-hover:opacity-100'}`}>
     {/* Copy, Delete, Regenerate buttons */}
   </div>
   ```

6. Ensure the existing Copy button (handleCopy) continues to work unchanged — the toast feedback is already working
7. Test:
   - Desktop: Hover over message → buttons appear → hover away → buttons disappear
   - Mobile: Tap message → buttons appear → wait 4sec → buttons fade
   - Multiple taps in succession → previous timeout is cleared, new 4sec starts

**Common pitfall (from RESEARCH.md):** If timeout is not cleared before starting a new one, buttons flicker. Use the pattern above to avoid this.

**Why 4 seconds:** Gives user time to read and interact with buttons without permanently cluttering the UI.

**Reference:** RESEARCH.md Pattern 2 (lines 151-199) shows the exact implementation approach.
