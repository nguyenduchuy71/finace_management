  <files>src/utils/dates.ts, src/utils/dates.test.ts</files>
  <action>
Extend `src/utils/dates.ts` with two new utility functions for delta calculation:

1. **calculateMonthDelta(currentAmount: number, previousAmount: number): number | null**
   - Returns percentage change: ((current - previous) / previous) * 100
   - Returns Math.round() result (whole number percentage)
   - Returns null if previousAmount <= 0 (cannot calculate % change from zero or negative baseline)
   - Edge case: negative to positive delta returns negative sign (e.g., -100 → 0 is "down", correctly labeled)

2. **getPreviousMonthDateRange(dateFrom: string | null, dateTo: string | null): { prevFrom: string; prevTo: string }**
   - If dateFrom provided: use it as reference to determine "current month"
   - If dateFrom null: use today's date in Vietnam timezone (UTC+7) as reference
   - Return previous month's first and last day (ISO format YYYY-MM-DD)
   - Handle edge cases:
     * January → December of previous year (year boundary)
     * Month-end transitions (Feb 28/29 → Jan 31, etc.)
   - MUST use TZDate from @date-fns/tz with VN_TZ='Asia/Ho_Chi_Minh'
   - Use date-fns: subMonths(), startOfMonth(), endOfMonth(), format()

Add tests in `src/utils/dates.test.ts`:

- Test getPreviousMonthDateRange: Jan 15 → Dec 1–31, Dec 15 → Nov 1–30, Feb 15 → Jan 1–31 (year boundary and month boundary cases)
- Test timezone: if using Western timezone offset, Vietnam UTC+7 month boundaries should still match expected dates
