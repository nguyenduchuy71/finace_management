<context>
## FilterStore State
Current filter state available via Zustand selectors:
- `accountId?: string` — selected bank account
- `cardId?: string` — selected credit card
- `dateFrom?: string | null` — start of date range
- `dateTo?: string | null` — end of date range
- `txType?: 'all' | 'income' | 'expense'` — transaction type filter
- `category?: Category | 'all'` — category filter
- `search?: string` — search query (merchant/description)

`useFilterParams()` hook returns all of above as object (used in existing FilterBar components).

## ExportButton Responsibilities
1. Capture current filter state via useFilterStore selectors
2. Determine which accountOrCardId is active (accountId ?? cardId)
3. On click: call exportTransactions() with accountOrCardId and all filters
4. Transform result via formatTransactionForCSV() and Papa.unparse()
5. Call downloadCSV() with generated CSV string and filename
6. Show loading state during fetch
7. Show success/error toast feedback

## Button Placement
Add ExportButton to FilterBar after ResetFilter button (logical grouping: filter controls + export).
Placement: right side of FilterBar, after ResetFilter, before or alongside other action buttons.

## Filename Format
transactions-YYYY-MM-DD.csv where date is today in Vietnam timezone.
Example: transactions-2026-03-09.csv

## Error Handling
- Network error: toast.error('Tải xuống thất bại — Vui lòng thử lại')
- Validation error: toast.error('Dữ liệu không hợp lệ')
- Generic error: toast.error('Tải xuống thất bại')
- Success: toast.success('Đã tải xuống CSV')
</context>
