<task type="auto">
  <name>Task 1: Create vercel.json with build and deployment configuration</name>
  <files>vercel.json</files>
  <action>
Create vercel.json in project root with Vercel-specific configuration:

```json
{
  "buildCommand": "tsc -b && vite build",
  "outputDirectory": "dist",
  "installCommand": "npm ci",
  "env": {
    "VITE_ENABLE_MSW": "false"
  },
  "envPrefix": "VITE_",
  "regions": ["sfo1"],
  "functions": {},
  "git": {
    "deploymentEnabled": {
      "main": true,
      "preview": true
    }
  }
}
```

Configuration details:
- `buildCommand`: Runs TypeScript build + Vite build (matches package.json build script)
- `outputDirectory`: dist/ is Vite's default build output
- `installCommand`: Use npm ci for reproducible installs (not npm install)
- `env`: Set VITE_ENABLE_MSW=false for production (MSW not needed, just real API)
- `envPrefix`: Vite uses VITE_ prefix for client-side environment variables
- `regions`: Deploy to US West (sfo1) for low latency; can be customized
- `git.deploymentEnabled`: Enable auto-deploy for main branch and preview for PRs

Rationale: Vercel reads vercel.json to determine build command, output directory, and environment variables. This configuration ensures the app builds and runs correctly in the Vercel environment.
  </action>
  <verify>
- vercel.json exists in project root
- JSON is valid (no syntax errors)
- `npm run build` matches buildCommand
- dist/ is correct outputDirectory (from vite.config.ts)
  </verify>
  <done>vercel.json created with build command, output directory, and environment configuration</done>
</task>
