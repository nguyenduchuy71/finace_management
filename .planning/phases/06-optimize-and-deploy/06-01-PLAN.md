  <name>Task 1: Configure Vite manual chunk splitting and tree-shaking</name>
  <files>vite.config.ts</files>
  <action>
Update vite.config.ts to enable aggressive code splitting:

1. Add `build.rollupOptions.output.manualChunks` to split:
   - `recharts` library into separate chunk (loaded only on DashboardPage)
   - `@anthropic-ai/sdk` into separate chunk (loaded only when chatbot is used)
   - `react-markdown` into separate chunk (loaded with chatbot)
   - Keep @tanstack/react-query in main bundle (used on all pages)

2. Add `build.rollupOptions.output.entryFileNames` and `chunkFileNames` for consistent naming

3. Verify tree-shaking is enabled (default in Vite, but set `build.minify: 'terser'`)

4. Set `build.chunkSizeWarningLimit: 1000` to suppress warnings for intentional large chunks

Example structure:
```typescript
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        recharts: ['recharts'],
        anthropic: ['@anthropic-ai/sdk'],
        markdown: ['react-markdown'],
      },
      entryFileNames: 'assets/[name]-[hash].js',
      chunkFileNames: 'assets/[name]-[hash].js',
    },
  },
  chunkSizeWarningLimit: 1000,
  minify: 'terser',
}
```

Rationale: Manual chunk splitting ensures heavy libraries load only when needed. This is safer than relying on dynamic import() alone, which can have waterfall loading issues.
  </action>

- `dist/assets/` contains at least 4 separate JS files (main + 3 chunks)

  </verify>
